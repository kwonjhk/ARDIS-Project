Handwritten Digit Recognition with ARDIS Data

Jinhyun Kwon

Summary

This project created classication models for the handwritten digit images of the ARDIS dataset, which is publicly available on the ARDIS GitHub website (https://ardisdataset.github.io/ARDIS/). Specically, this project focused on analysis of the 7,600 handwritten digit images taken from the ARDIS III dataset.

The images were processed in R using the magick package, attened into rows, and compiled into a single dataframe to be exported as a CSV le. We created three datasets corresponding to images with three pixel resolutions: 12x12, 20x20, and 32x32, denoted as img_12, img_20, and img_32. Dimensional reduction was applied to the 32x32 image dataset with principal components analysis (PCA). The rst 286 principal components, which explained 90% of the variance, were extracted and stored in a separate dataset.

For each dataset, logistic regression, K nearest neighbor, linear support vector machine, random forest, and bootstrap aggregating models were trained and evaluated based on their test set misclassication rates. Grid searching by cross-validation was used to tune each model; tuning was completed with the caret packages train() function.

The best model for the img_12 dataset was the random forest model with 12 variables per tree; the test error for this model was 0.0367. The random forest model was also the best model for the img_20 dataset, giving a similar test error of 0.04. For the PCA data, the best model was the KNN model with K=1, which produced a test error of 0.0467.

Our results show that models trained on the data containing the lowest resolution images, the img_12 dataset, performed as well as or better than equivalent models trained on the img_20 and PCA data. This seems to suggest that, for digit recognition, lowering the image resolution (to a certain degree) can boost computation speed and eciency without signicant loss of information or accuracy.

Introduction

Perhaps one of the most familiar applications of machine learning is image recognition technology. Facial recognition technology is an oft-cited, yet especially controversial, example of the most practical, powerful applications of contemporary machine learning innovations. Computers have neither the eyes nor the complex brain structures that humans use to look at and recognize objects; to computers, images are just large matrices of numbers. Nevertheless, researchers have devised a miscellany of predictive techniques that, combined with exponential improvements in computing technology, has achieved a striking level of precision in computer vision.

This project explores an elementary implementation of computer image recognition: handwritten digit identication. More specically, the interest of this project is to apply some more basic statistical learning techniques to a handwritten historical digit data called ARDIS (Arkiv Digital Sweden). The ARDIS data is composed of JPG images of Arabic numeral samples extracted from over 15,000 dierent nineteenth and twentieth century Swedish church records; the records were maintained by a number of dierent priests, each possessing his own calligraphic idiosyncrasies. The data itself was gathered by the Department of Computer Science from the Blekinge Institute of Technology in Karlskrona, Sweden and remains publicly available in several formats on the ARDIS Github website (https://ardisdataset.github.io/ARDIS/). The dataset that this project analyzes is the ARDIS III dataset, which contains 7,600 handwritten digit images with minimal background noise760 images per digit (0-9).

Image Processing

The ARDIS GitHub supplies four dierent datasets, three of which include the digit images in varying formats and one of which contains pre-processed numeric data on the digit images. We decided that, for the purpose of this project, the ARDIS III dataset comprised of individual digit images with minimal noise strikes an optimal balance between convenience and complexitywhile we do not have to worry about cleaning up the images or identifying strings of digits, we do have to format these raw images as a numeric matrix to properly apply our classication methods. Processing these images ourselves also allowed us to consider how recognition accuracy suers at the cost of computational ease aorded by decreased pixel resolution.

R, by no means the best program for handling images, nonetheless provides sucient support for basic manipulation; the magick package, a wrapper of ImageMagick for R, includes the simple image reading, rescaling, and recoloring functions that we required. We wrote two functions, convert_to_row() and ardis_data(), that completed the entire process of reading, editing, and attening images into a single data set.

The convert_to_row() function takes two parameters: img, an image object, and res, an integer representing the image resolution. Given the resolution, this function re-scales the image, converts all the pixel colors to either # or #000000 (white or black), changes those color values to 1 or 0, and nally collapses the image matrix into a single row.

The ardis_data() function, given a 0-9 digit and resolution, applies the convert_to_row() function with the specied resolution to the 760 images of the given digit and gathers them into a single data frame.

The full 7,600 row datasets were created by applying the ardis_data() function ten times, once for each digit, and then exported as CSV les. In total, we made three datasets, img_12, img_20, and img_32, containing raw numeric data of 12x12, 20x20, and 32x32 pixel images, respectively.

The images in Figure 1 (below) illustrate how the conversion alters the original handwritten digit image. Changing the original color to black and white does not lead to much loss of information. For simple numbers like 1, increasing the pixel resolution does not markedly improve the clarity of its features; there is not much detail needed because the shape itself is just a straight line. For more complicated digits like 8, the extra detail from increasing the resolution leads to a somewhat more marked improvement in their identiability. Increasing the resolution from 12x12 to 20x20 is a more drastic change in the image than increasing from 20x20 to 32x32, which suggests that the models for the img_20 dataset probably perform about the same as the models for the img_32 dataset. The handwritten digits are identiable to the untrained human eye at even the lowest resolution; whether such distinctions can be made by a computer remains to be seen.

Principal Component Analysis

The img_32 dataset is composed of the converted 32x32 digit images and thus contains 1,024 covariates for 7,600 observations. Though the dimensions of the data do not present any immediate issues, training models and performing cross-validation would be overtly computationally expensive, at least for our technological resources. We therefore performed dimensional reduction on img_32 and trained models on that data instead.

Principal components analysis (PCA) on the img_32 data enabled us to reduce the number of predictors from 1,024 to 286, the number of principal components that explain about 90% of the predictor variance. However, the data is still "heavy" in the sense that the 286 columns now contain a wider range of values than just 0 or 1, as in the original converted pixel matrix.

Surprisingly, the img_12 dataset resulted in the model with the lowest test misclassication rate of 0.0367; the random forest models for the img_12 and img_20 datasets performed about the same on the test set. Moreover, the best model trained on the PCA dataset, the most information-heavy dataset, had the highest test error overall. The overall similarities in test set performance suggest that the increase in image resolution does not noticeably improve the handwritten digit identiability.

We assume for the linear SVMs that the boundaries for the data points is best approximated as linear, but, seeing as the linear SVMs resulted in the highest test error for all 3 datasets, such an assumption likely does not hold. While a polynomial or radial basis may have performed better, those benets would be realized only after a very long training period. We attempted to perform a grid search for a polynomial SVM t to the img_12 but discovered that it would take hours just for our 144 dimension feature space, which did not bode well for the higher dimensional, more complex img_20 and img_32 PCA datasets. Hence, we forewent the more complex SVMs. Certainly, given a faster, multi-core computer processor that allows for parallel processing across several cores, we would have trained and evaluated the test performance of SVMs with nonlinear kernels.

As noted earlier, a peculiar feature of the KNN models was that, for all three of our datasets, the best model was the single nearest neighbor model. This 1-NN model lies at one extreme of the bias-variance trade-o, and we would typically surmise that such a model is grossly overt to the training data and would consequently not give adequate test results. However, after examining the test errors, we found that the 1-NN consistently outperformed the other KNN models. Hence, contrary to our intuition, the 1-NN model is the best KNN model for our data.

The main issues encountered in the project were technical; as mentioned before, we were unable to perform grid searches for polynomial or radial SVMs because the computations would take too long. The grid search for the random forest models initially posed a similar challenge. With the caret packages train() function, we rst applied the standard rf algorithm (from the randomForest package) but found that the time it took to get the out-of-bag errors for each mtry values was too long. We then tried parRF method with the train() function, but this alternative tting method did not drastically improve the computation time. Finally, in our next attempt, the ranger algorithm generated the tuning results within an hour; it is for this reason that we chose to use the ranger algorithm for all our random forest/bagging model tting.

The image processing, though at rst unfamiliar, was surprisingly simple; the magick package provided all the basic functions we needed, and we simply had to loop through the images downloaded from the ARDIS GitHub. This looping process required some minor string manipulation to read in the dierent image les, but that manipulation was the extent of our issues with handling the raw digit images.

Function to convert image to single row given image object and resolution value.

#parameters are img object (use image_read from 'magick') convert_to_row <- function(img, res) {

img %>% #re-scale image

image_scale(paste(res, "x", res, "!", sep = "")) %>% #convert to black and white

image_convert(type = 'Bilevel') %>% image_raster() %>%

#change black to 0 and black to 1

mutate(col = ifelse(col == "#ffffffff", 1, 0)) %>% #collapse

pivot_wider(id_cols=c("x", "y"), names_from = c("x", "y"), values_from = "col") %>%

return()

}
